public array
# ify( mixed $value, boolean $include_protected_private_properties = FALSE )

Forces an initial array value (NOT a deep scan).

Forces an initial array value (NOT a deep scan).

Parameters

$value
mixed

$value Anything can be converted to an array.

$include_protected_private_properties
boolean

$include_protected_private_properties Optional. Defaults to FALSE. By default, we do NOT include protected/private properties from objects being converted into arrays (even if they were accessible, which they are in this case). That is, when an object is converted to an array, all of its protected/private properties are included in the array, regardless of visibility (because PHP assumes there is no security issue once converted to an array). However, this behavior does come as a bit of a surprise, and it goes against the way most other WebSharks™ utility methods work. In addition, PHP will prefix each protected/private property with NULL bytes and namespace\class paths, resulting in a counterintuitive array of elements containing difficult-to-access keys.

Therefore, this routine will exclude protected/private properties by default. This behavior can be modified by setting $include_protected_private_properties to TRUE. If this is TRUE, protected/private properties will be included. When/if included, this routine removes the NULL byte and namespace\class paths, making protected/private property keys easy to access. Static properties are NEVER considered by this routine, because static properties are ignored while typecasting objects to arrays in PHP.

Returns

array

Arrayified value. This forces an initial array value at all times. This automatically includes both scalars and resources too.


See

http://www.php.net/manual/en/language.types.array.php#language.types.array.casting

Compiles a new array of all $key elements (deeply).

Compiles a new array of all $key elements (deeply).

Parameters

$array
array

$array An input array to search in.

$keys
string|integer|array

$keys An array of key elements to compile. In other words, elements with one of these array keys, are what we're looking for. A string|integer is also accepted here (if only one key), and it's converted internally to an array.

$preserve_keys
boolean

$preserve_keys Optional. Defaults to a FALSE value. If this is TRUE, the return array WILL preserve numeric/associative keys, instead of forcing a numerically indexed array. This ALSO prevents duplicates in the return array, which may NOT be desirable in certain circumstances. Particularly when/if searching a multidimensional array (where keys could be found in multiple dimensions). In fact, in some cases, this could return data you did NOT want/expect, so please be cautious.

$search_dimensions
integer

$search_dimensions The number of dimensions to search. Defaults to -1 (infinite). If $preserve_keys is TRUE, consider setting this to a value of 1.

$___current_dimension
integer

$___current_dimension For internal use only; used in recursion.

Returns

array

The array of compiled key elements, else an empty array, if no key elements were found. By default, the return array will be indexed numerically (e.g. keys are NOT preserved here). If an associative array is preferred, please set $preserve_keys to a TRUE value, and please consider setting $search_dimensions to 1.

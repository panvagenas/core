# parse( string $url_uri_query_fragment, null|integer $component = NULL, null|integer $normalize = NULL )

Parses a URL (or a URI/query/fragment only) into an array.

Parses a URL (or a URI/query/fragment only) into an array.

Parameters

$url_uri_query_fragment
string

$url_uri_query_fragment A full URL; or a partial URI; or only a query string, or only a fragment. Any of these can be parsed here.

$component
null|integer

$component Same as PHP's parse_url() component. Defaults to NULL; which defaults to an internal value of -1 before we pass to PHP's parse_url().

$normalize
null|integer

$normalize A bitmask. Defaults to NULL (indicating a default bitmask). Defaults include: websharks_core_v000000_dev\fw_constants::url_scheme, websharks_core_v000000_dev\fw_constants::url_host, websharks_core_v000000_dev\fw_constants::url_path. However, we DO allow a trailing slash (even if path is being normalized by this parameter).

Returns

array|string|integer|null

If a component is requested, returns a string component (or an integer in the case of PHP_URL_PORT). If a specific component is NOT requested, this returns a full array, of all component values. Else, this returns NULL on any type of failure (even if a component was requested).


Throws

websharks_core_v000000_dev\exception

If invalid types are passed through arguments list.


Note

A query string or fragment MUST be prefixed with the appropriate delimiters. This is bad name=value (interpreted as path). This is good ?name=value (query string). This is bad anchor (interpreted as path). This is good #fragment (fragment).


Arrays returned by this method, will include a value for each component (a bit different from PHP's parse_url() function). We start with an array of defaults (i.e. all empty strings, and 0 for the port number). Components found in the URL are then merged into these default values. The array is also sorted by key (e.g. alphabetized).
